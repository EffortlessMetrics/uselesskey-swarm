# v0.10.0 Facade Release Smoke

This record binds the release-candidate facade examples to command-backed proof
without publishing v0.10.0.

## User Path Covered

The checkout smoke copies the external example projects to
`target/external-adoption-smoke/work/`, rewrites their `uselesskey` dependencies
to the local checkout, and runs their tests as a downstream user would.

Covered examples:

- `examples/external/rust-test-fixtures`;
- `examples/external/webhook-verifier`;
- `examples/external/oidc-jwks-validation`;
- `examples/external/oidc-test-server-validation`;
- `examples/external/tls-chain-validation`;
- `examples/external/webauthn-ceremony-validation`;
- `examples/external/pkcs11-mock-validation`;
- `examples/external/ssh-fixture-validation`;
- `examples/external/pgp-fixture-validation`;
- `examples/external/hmac-signature-validation`;
- `examples/external/jsonwebtoken-adapter-validation`;
- `examples/external/entropy-byte-fixtures`;
- `examples/external/ecdsa-fixture-validation`;
- `examples/external/ed25519-fixture-validation`.

The bounded matrix is enforced by the xtask
`external_adoption_library_examples_are_bounded` regression test.

## Proof Command

```bash
cargo xtask external-adoption-smoke --path . --library-examples
```

## Generated Evidence

The smoke writes reviewable evidence under:

```text
target/external-adoption-smoke/report.json
target/external-adoption-smoke/report.md
target/external-adoption-smoke/logs/
target/external-adoption-smoke/work/
```

The receipt includes one `external-example-<name>` step for each copied
downstream-style project.

## Boundaries

- No publish.
- No tag.
- No crates.io push.
- No GitHub release.
- No source sync.
- No claim that v0.10.0 is installable from crates.io before publication.
- No production security claim.
- No provider compatibility claim.
- No downstream verifier correctness claim.
