# v0.10.0 Package Dry Run

This record captures package-readiness validation for the v0.10.0 release
candidate without publishing, tagging, signing, or pushing to crates.io.

## Package Coupling Fixed

Registry-style `cargo publish --dry-run` verifies package tarballs against
published dependency versions. In this checkout, the RSA, ECDSA, and Ed25519
crates called an internal `uselesskey-core` macro that is present in the local
checkout but not in the already-published `uselesskey-core` 0.9.1 package that
Cargo resolves during dry-run verification.

This slice keeps the public keypair accessors unchanged while inlining those
accessors in the three feature crates. That removes package-time reliance on a
new core macro and preserves the release boundary.

## Proof Commands

```bash
cargo xtask publish-preflight
cargo xtask publish-check
cargo xtask no-blob
cargo test -p uselesskey-rsa --all-features
cargo test -p uselesskey-ecdsa --all-features
cargo test -p uselesskey-ed25519 --all-features
cargo xtask lint-fix --check --no-clippy
```

## Package Contents Boundary

The package dry-run validates packageability and package-source consistency. It
does not publish crates or prove registry availability for future versions.

Generated release/adoption evidence remains under `target/` and is not a
packaged source artifact.

## Boundaries

- No publish.
- No tag.
- No crates.io push.
- No GitHub release.
- No source sync.
- No production security claim.
- No provider compatibility claim.
- No downstream verifier correctness claim.
