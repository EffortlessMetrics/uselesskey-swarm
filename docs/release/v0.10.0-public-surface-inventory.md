# v0.10.0 Public Surface Inventory

This is the release-readiness inventory for v0.10.0. It maps release-facing
surfaces to the user command or snippet, proof, release risk, and owner.

This is readiness evidence only. It does not publish crates, create tags, push
to crates.io, create a GitHub release, sign artifacts, or move source-sync
authority out of `EffortlessMetrics/uselesskey`.

## Follow-Up Work Items

- `version-snippet-reconciliation` decides which `0.9.1` snippets remain current
  stable and which v0.10.0 snippets are release-candidate or post-publish
  updates.
- `installed-cli-release-smoke` proves the CLI path from the release-candidate
  checkout.
- `facade-release-smoke` proves downstream library examples from the
  release-candidate checkout.
- `package-dry-run` validates package contents before any publication handoff.
- `release-readiness-record` records proof, hosted CI, non-blockers, publish
  order, rollback, and explicit non-claims.

## Boundaries

- No publish.
- No tag.
- No crates.io push.
- No GitHub release.
- No signing.
- No source sync.
- No production security claim.
- No provider compatibility claim.
- No downstream verifier correctness claim.
