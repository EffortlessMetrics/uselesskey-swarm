## Decision Rule

Do not advertise `0.10.0` as installable until it is published.

Use these classifications:

| Classification | Meaning | Release-readiness action |
| --- | --- | --- |
| Current stable | Public install or dependency snippet that should continue to say `0.9.1` until v0.10.0 is published. | Leave unchanged in readiness PRs. Update only in version-bump/package PRs after explicit publish readiness. |
| Release candidate | A readiness-only command, record, or release note may mention v0.10.0 as a target candidate. | Keep as review context only; do not provide it as a copyable install/dependency snippet. |
| Package version | Workspace crate versions and path dependency version constraints. | Leave unchanged until explicit version-bump/package dry-run slice. |
| Historical | Prior release evidence, learning, closeout, or audit docs. | Leave unchanged. |
| Internal path dependency | `path = ... version = "0.9.1"` inside workspace manifests. | Leave unchanged until version-bump handles package metadata coherently. |
